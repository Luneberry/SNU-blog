<h1>Augment Flip 참사 — Signed 메타 피처를 뒤집지 않았다</h1>

<p style="color:#6b7280; font-size:0.95em;">EXP 053에서 augment_flip을 켰더니 HL F1이 0.838에서 0.224로 붕괴했습니다. 원인은 좌우반전 시 부호를 바꿔야 하는 메타 피처를 그대로 두었기 때문입니다.</p>

<hr/>

<h2>1. 무슨 일이 있었나</h2>

<p>EXP 053은 augment_flip=OFF 상태로 composite 0.846을 기록했습니다. 좌우반전 augmentation으로 학습 데이터를 두 배 늘리면 성능이 더 오를 것이라는 기대로 EXP 054를 설계했습니다.</p>

<h2>2. 원인 분석</h2>

<p>좌우반전 augmentation은 좌표를 수평으로 뒤집은 뒤 HL↔HR 라벨도 교환합니다. 이 과정에서 <strong>방향성을 인코딩한 피처는 부호를 함께 반전해야 합니다.</strong></p>

<p>EXP 053의 meta 19ch 안에는 두 가지 유형이 섞여 있었습니다.</p>

<p><code>pelvis_min_diff</code>는 예를 들어 HL 파행이면 왼쪽 뒷다리 착지 때 골반이 더 내려오지 못해 양(+)의 값을 갖습니다. 좌우를 뒤집어 HR처럼 취급하면 이 부호가 (−)가 되어야 하는데, flip 코드에서 좌표 반전만 하고 이 채널은 그대로 뒀습니다.</p>

<p>결과적으로 "HR 라벨 + HL 비대칭 부호(미반전)"인 flip 샘플이 학습에 섞이면서, 모델이 HL 파행의 신호를 전혀 학습할 수 없게 됐습니다. HL을 제대로 구분하지 못하면서 HL↔HR 혼동이 폭증, HL F1이 0.838에서 0.224로 붕괴했습니다.</p>

<h2>3. 향후 수정 방향</h2>

<p>flip augmentation을 다시 적용하려면 코드에 부호 반전 처리를 추가해야 합니다.</p>

<pre style="background:#f3f4f6; padding:12px; border-radius:8px; font-size:0.9em; overflow-x:auto;"><code>META_NEGATE_ON_FLIP = [59, 62, 63]  # head_min_diff, pelvis_min_diff, pelvis_max_diff

def flip_with_meta_correction(window):
    window = flip_lr_coords(window)       # 좌표 반전
    window = swap_hl_hr_label(window)     # 라벨 교환
    window[:, META_NEGATE_ON_FLIP] *= -1  # signed 피처 부호 반전
    return window</code></pre>

<p>이 수정이 없으면 augment_flip은 단순히 학습 데이터를 두 배로 늘리는 게 아니라, 모순된 신호를 주입하는 셈입니다.</p>

<p style="background:#fef2f2; padding:12px; border-radius:8px;"><strong>교훈:</strong> 좌우반전 augmentation은 좌표만의 문제가 아닙니다. 피처 하나하나의 물리적 의미를 먼저 파악하고, 방향성을 가진 값은 반드시 함께 처리해야 합니다. 이를 확인하지 않은 채 flip을 적용하면 augmentation이 오히려 학습을 방해하는 독이 됩니다.</p>
