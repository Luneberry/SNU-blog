<p>hip 불일치의 원인: GREATER_TROCHANTER / ILIAC_CREST 키포인트 위치 추정 오차가 커서 고관절 각도 신호 자체가 노이즈에 묻힙니다. 관절별 pose 모델 신뢰도가 다름을 확인.</p>

<p>통계적으로 유의한 피처 (ANOVA, n=787):</p>
<ul>
<li><strong>상위 7위</strong>: Front_SI / head_amp / head_H1_H2_ratio / pelvis_min_diff / Hind_SI / pelvis_amp / head_min_diff — 기존 meta 7ch 독점</li>
<li><strong>8~12위에 관절 피처 진입</strong>: RB_tarsal_max, RB_hip_max, LB_tarsal_max, LB_hip_min, LB_tarsal_min</li>
<li><strong>severity 회귀 |r| 상위</strong>: RB_tarsal_max (−0.251), RB_hip_max (−0.222), LB_stifle_rom (−0.209), LF_shoulder_max (−0.182)</li>
</ul>

<p>관절 피처는 기존 meta와 독립축(|r| 0.15~0.25)으로, 추가 시 정보량이 늘어납니다. 이것이 19ch meta 후보가 된 근거입니다 (기존 7ch + 12ch 관절).</p>

<h2>3. 통합 데이터셋 — load_split() 단일 진입점</h2>

<p>각 프로젝트가 서로 다른 경로에서 데이터를 읽던 구조를 통합했습니다. <code>dog/data/</code>가 정본이 되고, 기존 경로들은 심볼릭 링크로 호환성을 유지합니다.</p>

<pre style="background:#f3f4f6; padding:12px 16px; border-radius:4px; overflow-x:auto; font-size:0.9em;"><code>from data.loader import load_split

# 단일파행 3-class (HL/HR/Normal)
train, test = load_split('category2_single', version='v2')

# 신경계 vs 정형외과 (2-class)
train, test = load_split('category1_neu', fold=0, label_scheme='binary')

# 관절 각도 meta 포함 리치 dict
train, test = load_split('category2_single', version='v2', rich=True)</code></pre>

<p>카테고리 구성: <code>category1_neu</code> (183환자, 2098 clips) / <code>category2_single</code> (HL/HR/Normal 3-class) / <code>category3_multi</code> (복합 파행, 빌드 보류) / <code>category4_graded</code> (중증도)</p>

<p>이 인프라 덕분에 이후 P3·P4 실험 코드가 데이터 경로를 하드코딩하지 않아도 됩니다.</p>
