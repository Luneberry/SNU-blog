<h3>주요 변경의 의미</h3>

<p><strong>YOLO26 vs RTMt:</strong> 관절 detection 모델이 변경되면서, 동일한 영상에서 추출되는 포즈 데이터 자체가 달라졌습니다. YOLO26은 더 최신의 detection 모델로, 관절 좌표의 정확도와 안정성에 차이가 있을 수 있습니다.</p>

<p><strong>환자 단위 분리:</strong> 이전에는 테스트 환자를 수동으로 지정했으나, 현재는 <code>splitter.py</code>의 <code>chart_number</code> 전략을 사용합니다. 엑셀에 등록된 차트번호(영상 보유 환자)를 테스트셋으로 자동 분리하여, 같은 개체의 클립이 train/test에 동시에 나타나지 않도록 보장합니다.</p>

<p><strong>FrameValidator:</strong> 포즈 데이터 로딩 시 프레임 단위로 품질 검증을 수행합니다. NaN 값, detection score, body ratio 이상치, hampel 필터 등 5가지 검증을 통해 노이즈 프레임을 제거합니다.</p>

<h2>4. 성능 향상 요인 분석</h2>

<p>모델이 동일하므로, 성능 향상의 원인은 <strong>데이터 파이프라인 개선</strong>에 있습니다.</p>

<div style="background:#f0f9ff; padding:16px; border-radius:8px; margin:16px 0;">
<h3 style="margin-top:0;">요인 1: Detection 데이터 변경 (RTMt → YOLO26)</h3>
<p>관절 detection 모델이 변경되면서 포즈 데이터 자체의 품질이 달라졌습니다. YOLO26은 더 최근에 학습된 모델로, 관절 좌표의 정확도가 개선되었을 가능성이 높습니다. 입력 데이터의 품질이 직접적으로 분류 성능에 영향을 미치는 것을 확인할 수 있습니다.</p>
</div>

<div style="background:#f0fdf4; padding:16px; border-radius:8px; margin:16px 0;">
<h3 style="margin-top:0;">요인 2: 데이터 파이프라인 체계화</h3>
<p>이전에는 클립 메타데이터와 테스트셋을 수동으로 구성했으나, 현재는 Project 0의 자동화된 파이프라인(<code>clip_data_auto.json</code>)을 사용합니다. <code>chart_number</code> 전략으로 환자 단위 분리가 보장되며, FrameValidator를 통한 프레임 품질 검증이 추가되었습니다.</p>
</div>

<div style="background:#fefce8; padding:16px; border-radius:8px; margin:16px 0;">
<h3 style="margin-top:0;">요인 3: 학습 재현성</h3>
<p><code>seed=42</code>로 고정하여 학습 재현성을 확보했습니다. 이전 실험에서는 seed가 고정되지 않아, 학습 랜덤성에 의한 성능 변동이 있었을 수 있습니다. 다만, seed 고정만으로 +16.6%p 향상을 설명하기는 어려우며, detection 데이터 변경이 주된 요인으로 판단됩니다.</p>
</div>

<h2>5. 시사점 및 향후 계획</h2>

<h3 style="margin-top:0;">핵심 시사점</h3>

<li><strong>"모델보다 데이터":</strong> 동일한 LSTM 모델에서 데이터 파이프라인 개선만으로 54.4% → 71%를 달성. 모델 구조 변경 없이도 입력 데이터 품질이 성능을 크게 좌우한다는 것을 실증.</li>
